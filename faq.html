Q. Has Lilac been released yet?
A. No, Lilac is still in very early develoment and has no planned release date yet.

Q. Can I build Lilac myself?
A. Yep. Use Git and clone "https://github.com/lilacide/lilacide.git", then run "npm i" (requires NodeJS) in the folder to install the dependencies. Finally, run "npm test" or "npm start" to view the correct usages.

Q. I get a "node-gyp" error when running "npm i". How do I fix this?
A. Make sure your version of Python is up to date. On Windows/macOS, some dependencies may need additional developer tools. On Windows, you can download and install the Visual Studio 2017 Build Tools from here or at the mirror. On macOS, you will need the developer tools from Xcode.

Q. Is the source code of this website public?
A. Yes, it is public on GitHub at "https://github.com/lilacide/lilacide.github.io".

Q. Why is the sidebar pure white?
A. Your OS may not support vibrancy. If you're using NodeJS, run "npm run electron-reduced-transparency" in a terminal. If you're using the precompiled executable, run "lilac.exe --reduce-transparency" instead.

Q. My sidebar is opaque, but my OS supports vibrancy. Can I fix this?
A. Lilac will try to automatically detect if your OS is compatible with the Electron window and Vibrancy. If it doesn't detect any compatibility with window vibrancy, it will automaatically run in reduced transparency mode. You can still try to force transparency by running this for NodeJS: "npm run electron-forced-transparency" or this for the precompiled executable: "lilac.exe --force-transparency", but there's no guarantee that it will work. If it fails, your sidebar will appear with a pure white backgroud instead of being transparent.
